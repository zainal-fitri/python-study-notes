Identifiers & Syntax

Naming Rules (Identifiers)

  • Can contain letters (a-z, A-Z), numbers (0-9), and underscores (_).
  • Must start with a letter or an underscore.
  • Cannot start with a number.
  • Cannot contain spaces or special symbols like @, $, %.
  • Case Sensitive: Age and age are different.
Valid: my_var, _count, var2
Invalid: 2var, my-var, my var

1.2 Python Data Structures

Difference vs. Data Types: Data Types (int, float) are atomic (single values). Data Structures are collections that store multiple items. Strings are considered collections of characters!

" "

Indexing & Slicing

Format: [start : stop : step]. Note: Stop is NOT inclusive.

s = "PYTHON"
print(s[0]) # 'P'
print(s[-1]) # 'N' (Negative Index)
print(s[0:2]) # 'PY' (Stops before 2)

Key Methods

  • len(s) -> Count characters
  • s.upper() -> "PYTHON"
  • s.lower() -> "python"
  • s.title() -> "Python" (Capitalize 1st letter)

Control Structures & Loops

1. Decision Making (Conditionals)

a. If Statement

Runs the block only if the condition is True.

age = 20
if age >= 18:
    print("Adult")

b. If...Else

Runs one block if True, another if False.

if age >= 18:
    print("Adult")
else:
    print("Minor")

c. If...Elif...Else

Checks multiple conditions in sequence.

if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
else:
    grade = "F"

2a. While Loop

Condition Based

Repeats a block of code as long as a condition is True.

count = 0
while count < 5:
    print(count)
    count += 1
Caution: Ensure the condition eventually becomes False to avoid infinite loops!

2b. For Loop

Iterator Based

Iterates over a sequence (list, string, or range).

# Iterate over list
fruits = ["Apple", "Banana"]
for fruit in fruits:
    print(fruit)
# Iterate using range()
for i in range(5):
    print(i) # 0, 1, 2, 3, 4

Basic Python Reference

Interpreted Language

Python code is not compiled into machine code all at once. Instead, it is processed at runtime by the Python Interpreter.

How it works:

  1. Source code (.py) is read.
  2. Compiled to Bytecode (.pyc).
  3. Executed by the Python Virtual Machine (PVM).

Pros & Cons:

  • Platform Independent (Runs anywhere)
  • Easier Debugging
  • Slower execution than C/C++

Dynamic Typing

In Python, variables are labels (references) to objects, not fixed containers. The "type" belongs to the object, not the variable name.

# Variable x can change types
x = 5 # x is pointing to an int
x = "Hello" # Now x points to a string

Note: Python is also Strongly Typed. This means it won't allow you to mix incompatible types (e.g., "1" + 2 will fail).

Operator Precedence & Associativity

Order Operator Description Associativity
1 (Highest) () Parentheses (Grouping) Left-to-Right
2 ** Exponentiation Right-to-Left
3 ~, +, - Unary plus/minus, bitwise NOT Right-to-Left
4 *, /, //, % Multiply, Divide, Modulo Left-to-Right
5 +, - Addition, Subtraction Left-to-Right
6 ==, !=, >, < Comparisons, Identity, Membership Left-to-Right
7 not Logical NOT Right-to-Left
8 (Lowest) and, or Logical AND, OR Left-to-Right
Note on Associativity: Most operators evaluate Left-to-Right (e.g., 10 - 5 - 2 is (10 - 5) - 2 = 3).
However, Exponents evaluate Right-to-Left: 2 ** 3 ** 2 is 2 ** (3 ** 2) = 2^9 = 512.
